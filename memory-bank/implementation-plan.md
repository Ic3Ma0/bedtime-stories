# 实现计划

> 规则：每步30-60分钟，无代码片段，含验证测试，分阶段推进

---

## Phase 1: Foundation（基础骨架）

### Step 1: 项目骨架搭建
- **动作**: 创建 `index.html`、`css/styles.css`、`js/` 目录及空模块文件，配置 Tailwind CDN 和 Lucide Icons CDN
- **验证**: 浏览器打开 `index.html`，页面正常显示标题，控制台无404错误，Tailwind工具类生效（如 `bg-blue-500` 能显示蓝色背景）
- **时间**: 30分钟

### Step 2: 主题系统与基础布局
- **动作**: 实现日间/夜间模式CSS变量，完成页面三区布局（顶部导航栏、中部故事阅读区、底部操作栏），支持移动端响应式
- **验证**: 
  - 切换夜间模式按钮，背景色从浅色变为深色，文字从深色变为浅色
  - 手机浏览器打开，阅读区宽度自适应，无横向滚动条
  - 刷新页面后主题偏好保持（localStorage记忆）
- **时间**: 45分钟

---

## Phase 2: Core（核心功能）

### Step 3: 预制故事库
- **动作**: 在 `js/story-data.js` 中编写20个轻悬疑/微恐怖短故事，每个故事包含标题、标签、时长、正文；确保主角为人类或超自然存在，非动物
- **验证**: 
  - 控制台执行 `console.log(stories.length)` 输出 20
  - 随机抽查5个故事，主角均非动物，且结局有反转或悬念
  - 每个故事字数在 500-1500 字之间（适合3-8分钟阅读）
- **时间**: 60分钟

### Step 4: 故事生成器与阅读渲染
- **动作**: 实现 `story-generator.js`（从预制库随机选取）和 `ui-controller.js`（将故事渲染为DIV排版阅读界面），支持"换一篇"按钮
- **验证**: 
  - 点击"换一篇"，阅读区内容更换为新故事，标题、正文、标签正确显示
  - 连续点击10次，无重复故事（或重复率 < 20%）
  - DIV排版符合中文阅读体验：字号18px+、行高1.8、段落间距舒适
- **时间**: 45分钟

### Step 5: 本地存储与收藏功能
- **动作**: 实现 `storage.js`（localStorage封装），支持收藏/取消收藏故事，收藏列表页面，阅读进度记忆
- **验证**: 
  - 收藏一个故事，刷新页面后收藏状态保持
  - 打开收藏列表，显示已收藏故事的标题
  - 阅读到故事中间位置，刷新后自动回到该位置
- **时间**: 45分钟

### Step 6: 语音朗读（TTS）
- **动作**: 实现 `speech.js`，封装 Web Speech API，支持播放/暂停/语速调节
- **验证**: 
  - 点击朗读按钮，浏览器开始语音播放当前故事
  - 点击暂停，语音停止；再次点击从暂停处继续
  - 调节语速滑块，语音播放速度明显变化
  - （如浏览器不支持TTS，友好提示"当前浏览器不支持语音朗读"）
- **时间**: 45分钟

---

## Phase 3: Polish（打磨与AI扩展）

### Step 7: AI故事生成集成
- **动作**: 在 `story-generator.js` 中接入 Gemini API，当用户输入API Key且在线时，调用AI生成新故事；添加API Key输入弹窗和"AI生成"开关
- **验证**: 
  - 输入有效Gemini API Key，点击"AI生成"，5秒内显示全新故事（非预制库内容）
  - 断开网络，点击生成，自动回退到预制库，无报错
  - 未输入API Key时，点击"AI生成"提示先配置
- **时间**: 60分钟

### Step 8: 细节打磨与体验优化
- **动作**: 添加加载动画、空状态提示、操作Toast反馈、键盘快捷键（空格朗读/ESC关闭弹窗）、故事长度筛选（短篇/中篇）
- **验证**: 
  - 点击"换一篇"时出现加载动画，内容加载后消失
  - 收藏故事时弹出"已收藏"提示，2秒后自动消失
  - 按空格键开始/暂停语音朗读
  - 筛选"短篇"后，只显示字数 < 800 的故事
- **时间**: 45分钟
