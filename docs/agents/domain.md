# Domain Docs

## Layout

Single-context. The repo has one global domain context.

## Files

- `CONTEXT.md` — Domain ubiquitous language and concepts (at repo root)
- `docs/adr/` — Architecture Decision Records

## Consumer rules

- Read `CONTEXT.md` before making naming decisions or architectural changes
- Check `docs/adr/` before proposing changes that contradict past decisions
- Update `CONTEXT.md` when new domain terms emerge
